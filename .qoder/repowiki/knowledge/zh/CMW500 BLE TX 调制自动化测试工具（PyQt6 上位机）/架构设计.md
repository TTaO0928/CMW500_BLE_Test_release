单仓库扁平结构，按职责拆分为五个顶层模块：
- `main.py`：程序入口，负责解析 `config.yaml`、兼容旧配置（`_normalize_config`）、根据 `--cli` 参数选择 CLI 或 GUI 模式，并统一全局异常兜底。
- `instrument_connection.py`：封装 VISA 资源管理，通过 `_build_resource_address` 动态拼接 LAN/GPIB/USB 地址，提供 `connect/disconnect/query/send_command/get_serial_number` 等 SCPI 通道。
- `test_executor.py`：编排 BLE TX 调制测试流程（`configure_instrument` → 遍历 Channel 0~39 → `measure_single_channel`），依据 `config.yaml` 中 `measurements` 的上下限计算 PASS/FAIL，并通过回调向 UI 推送日志、进度和逐信道结果。
- `gui_main.py`：PyQt6 主窗口 + `TestWorker(QThread)` 工作线程，使用 `pyqtSignal` 跨线程更新 QTableWidget、QProgressBar 和日志区；界面顶部用 `QStackedWidget` 切换 LAN/GPIB/USB 参数输入区。
- `data_exporter.py`：以 pandas+openpyxl 生成双 Sheet（"测试数据" + "测试摘要"）Excel，自动应用表头/边框/PASS-FAIL 着色与列宽自适应。
依赖方向单向：GUI/CLI → TestExecutor → InstrumentConnection；DataExporter 仅消费测试结果列表，无反向依赖。所有外部库（PyQt6、pyvisa、pandas、openpyxl、PyYAML）采用延迟导入以避免启动时闪退。
