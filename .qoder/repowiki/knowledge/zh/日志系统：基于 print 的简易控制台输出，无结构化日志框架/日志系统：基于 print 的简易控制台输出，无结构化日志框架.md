本仓库未实现正式的日志系统。全项目仅使用 Python 标准库 `print()` 进行控制台输出，未发现任何第三方日志框架（如 loguru、structlog）或标准库 logging 模块的使用痕迹。

- **输出方式**：所有信息通过 `print(f"[信息] ...")`、`print("[错误] ...")`、`print("[成功/失败] ...")` 等带中文标签的字符串直接打印到 stdout/stderr。
- **级别策略**：没有统一的日志级别管理，开发者自行在字符串前缀中区分「信息」「错误」「提示」「成功/失败」等语义，属于约定式而非机制式控制。
- **结构化字段**：无统一格式，消息多为自由文本拼接，不包含时间戳、线程 ID、模块名等元数据。
- **持久化与路由**：无文件写入、无多 sink 路由、无日志轮转；GUI 模式下的状态更新通过 PyQt6 信号槽刷新界面，命令行模式则直接输出到终端。
- **配置文件**：`config.yaml` 仅包含仪器连接参数与测试配置，不含任何日志相关设置。

由于本项目是面向本地桌面使用的单用户工具，且运行时长短、调试主要依赖实时 GUI 展示，因此未引入正式日志框架。若后续需要增强可观测性，建议引入 `logging` 或 `loguru`，并建立统一的 logger 初始化入口与级别策略。
